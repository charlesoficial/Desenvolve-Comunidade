## 🎯 Estratégia de Auth — DECISÃO PENDENTE

O frontend (`lib/auth.ts`) usa **Supabase GoTrue** direto (não nossa API Rails).
A coluna `password_digest` que criamos no Rails fica redundante.

**Opção A — Padronizar em Supabase Auth (recomendado, é o que P6/Circle faz)**
- Manter `lib/auth.ts` como está
- Remover endpoints `/auth/signup` e bcrypt do Rails
- ApplicationController fallback dev continua existindo
- Tabelas RLS no Supabase fazem o controle de acesso

**Opção B — Padronizar em Rails Auth**
- Reescrever `lib/auth.ts` para chamar nossa API Rails
- Remover dependência do GoTrue
- Manter bcrypt + endpoints já implementados

**Recomendação:** A. Mais alinhado com a stack Supabase, menos código pra manter.
